接口说明:
参数	类型	参数值	说明
sign	string	加密后字符串	32 位小写 md5(random+sn+secretKey)
random	string	随机字符串	16-32 位 小写字母 + 数字
sn	string	固定字符串	商户前缀，请登录管理后台查看
Content-Type	string	application/json	请求类型
所有接口都是 POST 协议进行请求，并且接口需要验签，验签方式为在请求头中设置4个参数
参数 sn 和 secretKey 请登录管理后台查看，如需申请请联系商务
random 为随机字符串，长度限制 16-32 位 小写字母 + 数字组合
sign 为 String sign=MD5(random+sn+secretKey) 的结果
数据返回格式统一 Json 格式
请求接口的 IP 需要添加白名单（管理后台可以自助添加）
所有数据时区为北京时间
建议所有接口的请求超时时间设置为 30 秒以上
请求返回正确示例：
{
	“code”: 10000, // 状态码，10000 成功，其他失败
	“msg”: “request succeeded”, // 消息
	“data”: null
}
请求返回错误示例：
{
	“code”: 10404, // 状态码，10000 成功，其他失败
	“msg”: “Signature verification failed”, // 消息
	“data”: null
}
